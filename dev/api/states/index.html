MPS Type Aliases

DynamicalCorrelators.FiniteNormalMPS — Type
const FiniteNormalMPS{C}

Type alias for a standard finite MPS with 2-leg physical tensors (virtual-physical; virtual). Used for zero-temperature ground states.

source
DynamicalCorrelators.FiniteSuperMPS — Type
const FiniteSuperMPS{C}

Type alias for a finite MPS with 3-leg physical tensors (virtual-physical-ancilla; virtual). Used for finite-temperature density matrix purification (thermofield double formalism).

source

Charged MPS

DynamicalCorrelators.chargedMPS — Function
chargedMPS(operator::AbstractTensorMap, state::FiniteNormalMPS, site::Integer)

Apply a local operator at site to a normal MPS by constructing a charged MPO and contracting it with the state. Returns the resulting MPS (e.g., c†ᵢ|ψ⟩).

source
chargedMPS(operator::AbstractTensorMap, state::FiniteNormalMPS, site₁::Integer, site₂::Integer)

Apply a two-site operator at site₁ and site₂ to a normal MPS.

source
chargedMPS(op::AbstractTensorMap{B,S,1,2}, mps::FiniteSuperMPS, site::Integer)

Apply a (1,2)-leg operator (1 codomain, 2 domain legs) to a super MPS at site. This inserts the operator into the physical-ancilla leg structure of the purified state. For sites after the operator site, the tensor legs are rearranged via braiding (τ).

source
chargedMPS(op::AbstractTensorMap{S,B,2,1}, mps::FiniteSuperMPS, site::Integer)

Apply a (2,1)-leg operator (2 codomain, 1 domain legs) to a super MPS at site. For sites before the operator site, the tensor legs are rearranged via braiding (τ).

source
chargedMPS(op::AbstractTensorMap{S,B,1,1}, mps::FiniteSuperMPS, site::Integer)

Apply a (1,1)-leg operator (diagonal, charge-neutral) to a super MPS at site. Only the tensor at the operator site is modified; all other tensors remain unchanged.

source
DynamicalCorrelators.identityMPS — Function
identityMPS(H::FiniteMPOHamiltonian)

Construct an identity MPS (purified infinite-temperature density matrix) from a Hamiltonian. Each site tensor is a BraidingTensor acting as an identity map on the physical space. This serves as the initial state for imaginary-time evolution to finite temperature.

source

Random MPS

DynamicalCorrelators.randFiniteMPS — Function
randFiniteMPS(elt, U1Irrep, U1Irrep, N; filling=(1,1), md=10, mz=nothing)

Create a random finite MPS with fℤ₂ × U(1) × U(1) symmetry (charge and spin conservation).

Arguments

  • elt: element type (e.g., Float64, ComplexF64).
  • N: number of sites.
  • filling: tuple (P, Q) defining the filling fraction P/Q. Requires N to be a multiple of 2Q.
  • md: bond dimension for each quantum number sector (default: 10).
  • mz: maximum charge quantum number range. If nothing, determined from filling.
source
randFiniteMPS(elt, SU2Irrep, U1Irrep, N; filling=(1,1), md=10, mz=nothing)

Create a random finite MPS with fℤ₂ × SU(2) × U(1) symmetry (spin-rotation and charge conservation).

Arguments

  • elt: element type.
  • N: number of sites.
  • filling: tuple (P, Q) defining filling P/Q.
  • md: bond dimension per sector (default: 10).
  • mz: maximum quantum number range. If nothing, determined from filling.
source
randFiniteMPS(elt, H::MPOHamiltonian; left=oneunit(...), right=oneunit(...))

Create a random finite MPS compatible with the given MPO Hamiltonian H. The virtual spaces are automatically inferred from the physical spaces and boundary conditions.

source
randFiniteMPS(elt, pspace, N; right=oneunit(pspace))

Create a random finite MPS with N sites, each having physical space pspace.
